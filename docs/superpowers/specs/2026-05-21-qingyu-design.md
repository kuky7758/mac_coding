# 轻语 — 设计规格

## 项目概述

一个最小可用的匿名短内容发布与浏览应用，纯前端实现，数据持久化在浏览器 Local Storage 中。单文件部署，复制即用。

## 功能范围

**包含：**
- 匿名发布纯文字内容（最多 280 字）
- 按时间倒序浏览已发布内容
- 实时字数统计
- 空状态提示
- 数据刷新后保留（Local Storage）

**明确不包含（超出最小可用范围）：**
- 用户身份/登录/注册
- 图片、链接、富文本
- 删除、编辑已发布内容
- 点赞、评论、转发
- 搜索、筛选、分页
- 多页面路由
- 后端服务

## 技术栈

- HTML5（单文件 `index.html`）
- CSS3（内嵌样式）
- JavaScript（原生，无框架、无构建工具）
- Local Storage API
- Git 版本管理

## 页面布局

单页应用，垂直三区块：

1. **顶部标题区**
   - 应用名："轻语"
   - 副标题："写下一句话，记录当下"

2. **输入区**
   - 多行文本框（textarea）
   - 右下角字数统计（`当前字数/280`）
   - 蓝色发布按钮
   - 整体卡片式容器

3. **时间线列表区**
   - 每条内容为一个卡片
   - 卡片内：文字内容 + 发布时间（`YYYY-MM-DD HH:mm` 格式）
   - 按发布时间倒序排列，最新在上
   - 无数据时显示空状态文案

## 数据结构

Local Storage 键名：`qingyu_posts`

```js
[
  {
    id: 1716280000000,      // 时间戳，兼作唯一标识
    content: "文字内容",     // 用户输入，字符串
    timestamp: 1716280000000 // 发布时间，毫秒时间戳
  }
]
```

- 使用 `Date.now()` 生成 id 和 timestamp
- 读取时 `JSON.parse`，写入时 `JSON.stringify`
- 渲染前按 `timestamp` 降序排列

## 交互逻辑

### 页面初始化
1. 从 `localStorage.getItem('qingyu_posts')` 读取数据
2. 解析为数组，若不存在则初始化为空数组
3. 按 `timestamp` 降序排列
4. 调用渲染函数生成 DOM

### 发布流程
1. 监听发布按钮点击事件
2. 获取 textarea 内容并 `trim()`
3. 空内容：不做任何操作（按钮可提供视觉反馈）
4. 非空内容：
   - 创建对象 `{id: Date.now(), content, timestamp: Date.now()}`
   - `unshift` 到数组开头
   - `JSON.stringify` 后 `localStorage.setItem`
   - 重新渲染列表
   - 清空 textarea
   - 重置字数统计

### 实时字数统计
- 监听 textarea `input` 事件
- 实时计算 `textarea.value.length`
- 更新右下角计数显示
- 超出 280 字时数字标红（不禁止发布）

## 边界情况

| 场景 | 处理方式 |
|---|---|
| 首次访问，无数据 | 列表区显示"还没有内容，写下一句话吧" |
| 发布空内容 | 忽略操作，发布按钮可闪动提示 |
| Local Storage 满 | `try-catch` 包裹写入逻辑，`alert('存储空间不足，请清理后重试')` |
| 手动清除浏览器数据 | 内容丢失，符合预期（本地存储特性） |

## 验收标准

1. 在浏览器中打开 `index.html` 可正常使用
2. 发布内容后立即出现在列表顶部
3. 刷新页面后内容不丢失
4. 关闭浏览器重新打开后内容不丢失
5. 空状态文案在无数据时正确显示
6. 字数统计实时更新且准确
7. 代码结构清晰，单文件即可运行
